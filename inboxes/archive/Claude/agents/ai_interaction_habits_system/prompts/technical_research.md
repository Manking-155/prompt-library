# Technical Research Prompt Templates

Tập hợp các prompt patterns hiệu quả cho technical research và validation.

## 🔍 Research Investigation Patterns

### Technology Evaluation Template
```
Analyze [technology/tool/framework] for [specific use case]:

Context:
- Current tech stack: [Flutter, Python, MySQL, etc.]
- Project requirements: [specific needs]
- Constraints: [budget, timeline, complexity]

Please provide:
1. Technical capabilities analysis
2. Integration complexity assessment
3. Performance implications
4. Cost-benefit analysis
5. Alternative comparisons
6. Implementation recommendations

Format response with clear sections and actionable insights.
```

### Market Research Pattern
```
Research [market/industry/trend] for [business context]:

Focus Areas:
- Market size and growth potential in Vietnam
- Key players and competitive landscape
- Cultural considerations and local preferences
- Technical implementation challenges
- Revenue model opportunities

Provide:
1. Market overview with data points
2. Competitive analysis table
3. Opportunity assessment
4. Risk factors identification
5. Go-to-market recommendations

Include Vietnamese market specifics where relevant.
```

### Problem-Solution Validation
```
Validate solution for [problem statement]:

Problem Context:
- Target users: [demographics]
- Current pain points: [specific issues]
- Existing solutions: [alternatives]
- Market gap: [opportunity]

Validation Framework:
1. Problem severity assessment
2. Solution feasibility analysis
3. Market fit evaluation
4. Technical implementation path
5. Business viability check
6. Risk mitigation strategies

Provide structured assessment with confidence scores.
```

## 💻 Code Generation Patterns

### Flutter Feature Implementation
```
Generate Flutter implementation for [feature]:

Requirements:
- Feature description: [detailed requirements]
- Architecture: Clean Architecture with Riverpod
- Database: Hive for local, MySQL for backend
- AI integration: [Claude API/offline models]

Generate:
1. Project structure
2. Domain entities and use cases
3. Data models and repositories
4. Presentation widgets and providers
5. Unit test templates
6. Implementation checklist

Follow established patterns from .agent documentation.
```

### Database Design Pattern
```
Design database schema for [application]:

Context:
- Application type: [mobile app/web service]
- Data requirements: [specific data needs]
- Scale expectations: [user volume, data size]
- Technology: MySQL on Oracle Cloud

Provide:
1. Entity relationship diagram
2. Table schemas with indexes
3. Migration scripts
4. Performance optimization suggestions
5. Backup and scaling strategies
6. Security considerations

Include both SQL and corresponding Hive models for offline sync.
```

### AI Integration Pattern
```
Design AI integration for [specific feature]:

Requirements:
- AI capability needed: [specific AI task]
- Platform: Flutter mobile app
- Performance requirements: [response time, accuracy]
- Budget constraints: [API costs]

Provide:
1. AI service comparison (Claude API vs offline models)
2. Integration architecture
3. Implementation code templates
4. Error handling strategies
5. Performance optimization techniques
6. Cost analysis and recommendations

Include both online and offline implementation options.
```

## 📊 Business Analysis Patterns

### Revenue Model Analysis
```
Analyze revenue models for [product/service]:

Product Context:
- Target market: [Vietnamese users/international]
- User demographics: [age, income, behavior]
- Value proposition: [core benefits]
- Competition: [existing solutions]

Evaluate:
1. Freemium vs subscription vs one-time purchase
2. Pricing strategy recommendations
3. Revenue projections with assumptions
4. Customer acquisition cost analysis
5. Lifetime value calculations
6. Monetization optimization suggestions

Focus on Vietnamese market dynamics and purchasing power.
```

### Competitive Analysis Template
```
Conduct competitive analysis for [product category]:

Analysis Scope:
- Direct competitors: [similar solutions]
- Indirect competitors: [alternative approaches]
- Market positioning: [differentiation opportunities]
- Geographic focus: [Vietnam + international]

Provide:
1. Competitor feature comparison matrix
2. Pricing analysis table
3. Strengths and weaknesses assessment
4. Market positioning map
5. Differentiation opportunities
6. Strategic recommendations

Include local Vietnamese competitors and cultural factors.
```

## 🎯 Learning and Development Patterns

### Skill Gap Analysis
```
Assess skill development for [technology/domain]:

- Technical skills: [Flutter, Python, Database design]
- Domain knowledge: [Real estate, AI, Mobile development]
- Business skills: [Strategy, Marketing, Product management]

Target Goals:
- Professional objectives: [solo developer growth]
- Technical advancement: [new technologies to learn]
- Business development: [entrepreneurship skills]

Provide:
1. Skill gap identification
2. Learning priority ranking
3. Resource recommendations (courses, books, projects)
4. Timeline for skill acquisition
5. Practice project suggestions
```
