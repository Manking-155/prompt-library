## 🚀 Getting Started

### For New Users
1. Đọc tài liệu trong **[System](system/)** để hiểu kiến trúc (15 phút)
2. Xem **[SOPs](sops/)** để hiểu các quy trình (15 phút)
3. Kiểm tra **[Templates](templates/)** để xem các template có sẵn

### For Starting a New Project
1. Kiểm tra **[Tasks](tasks/)** để xem PRDs và plans hiện có
2. Sử dụng templates từ **[Templates](templates/)** để triển khai
3. Làm theo các SOP từ **[SOPs](sops/)** cho quy trình

## 🔄 Updating Documentation

Sau khi triển khai feature hoặc process mới:
1. Thêm examples vào template nếu phù hợp
2. Cập nhật SOP nếu workflow thay đổi
3. Tạo tasks/PRD cho features mới
4. Cập nhật system documentation

## 📝 Documentation Standards

- Tất cả tài liệu bằng tiếng Việt
- Cung cấp examples và templates thực tế
- Bao gồm troubleshooting sections
- Link đến tài liệu liên quan
- Giữ examples đơn giản và tập trung

## ✅ Pre-Implementation Checklist

Trước khi bắt đầu bất kỳ implementation nào:
- [ ] Requirements rõ ràng
- [ ] Đã đọc tài liệu liên quan
- [ ] Business model phù hợp với mục tiêu
- [ ] Financial planning đã được thiết lập
- [ ] Hiểu context thị trường Việt Nam
- [ ] Biết project của bạn fit vào architecture như thế nào
