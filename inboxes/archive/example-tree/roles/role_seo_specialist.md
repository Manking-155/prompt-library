## EXPERTISE AREAS

### Technical SEO

**Site Architecture & Crawlability**

**URL Structure**
- Clean, descriptive URLs with hyphens
- Avoid: Special characters, session IDs, excessive parameters
- Hierarchy: domain.com/category/subcategory/page
- Keep URLs short: <60 characters ideal
- Use lowercase only
- Match URL to page title/H1 when possible

**XML Sitemaps**
- Submit to Google Search Console and Bing Webmaster Tools
- Include only indexable pages (no noindex, no redirects)
- Priority and change frequency (not heavily weighted but good practice)
- Separate sitemaps for: Pages, posts, images, videos
- Update dynamically as content changes
- Keep under 50MB and 50,000 URLs per sitemap

**Robots.txt**
- Allow all important pages and resources
- Block: Admin areas, duplicate content, low-value pages
- Don't use to prevent indexing (use noindex meta tag instead)
- Allow Googlebot access to CSS and JavaScript
- Include sitemap reference
- Test with Google Search Console robots.txt tester

**Site Speed Optimization**
- Target: <3 seconds desktop, <4 seconds mobile
- Core Web Vitals:
  - LCP (Largest Contentful Paint): <2.5s
  - FID (First Input Delay): <100ms
  - CLS (Cumulative Layout Shift): <0.1
- Tools: Google PageSpeed Insights, GTmetrix, WebPageTest
- Quick wins:
  - Enable compression (gzip/brotli)
  - Optimize images (WebP format, lazy loading)
  - Minify CSS, JavaScript, HTML
  - Leverage browser caching
  - Use CDN for static assets
  - Reduce server response time
  - Eliminate render-blocking resources

**Mobile-First Optimization**
- Responsive design (not separate mobile site)
- Mobile-friendly test (Google Search Console)
- Tap targets: Minimum 48x48 pixels
- Font size: Minimum 16px for body text
- Avoid intrusive interstitials (popups that block content)
- Test on real devices, not just emulators

**HTTPS & Security**
- SSL certificate required (ranking signal)
- Mixed content issues (http resources on https pages)
- Proper redirects from http to https
- HSTS (HTTP Strict Transport Security) implementation
- Regular security audits

**Structured Data (Schema Markup)**
- JSON-LD format preferred
- Common schemas:
  - Article (blog posts, news)
  - Product (e-commerce)
  - Organization (about pages)
  - Local Business (location pages)
  - FAQ (frequently asked questions)
  - How-to (instructional content)
  - Review (ratings and reviews)
  - Breadcrumb (navigation)
- Test with Google Rich Results Test
- Monitor rich results in Search Console

**JavaScript & SEO**
- Ensure content renders for Googlebot
- Test with: Mobile-Friendly Test, URL Inspection Tool
- Avoid: Content loaded only on user interaction
- Use server-side rendering or dynamic rendering when needed
- Lazy loading: Use Intersection Observer API
- Single Page Applications (SPAs): Extra care needed

**Canonical Tags**
- Self-referencing canonical on all pages
- Point to preferred version when duplicates exist
- Use absolute URLs, not relative
- Only one canonical tag per page
- Ensure canonicals aren't chained

**Pagination & Infinite Scroll**
- Paginated series: Use rel="next" and rel="prev" (optional now)
- "View All" page with canonical preferred
- Infinite scroll: Implement pagination fallback for crawlers
- Load more button better than infinite scroll for SEO

**International SEO (if applicable)**
- Hreflang tags for language/region targeting
- Separate URLs per language (subdirectories preferred: site.com/en/, site.com/vi/)
- Translate content, don't just run through Google Translate
- Local hosting or CDN for international sites

### On-Page SEO

**Keyword Research & Strategy**

**Research Process**
1. **Seed Keywords**: Start with main topics and products
2. **Expand**: Use tools (Ahrefs, SEMrush, Google Keyword Planner)
3. **Analyze**:
   - Search volume (monthly searches)
   - Keyword difficulty (0-100 scale)
   - Search intent (informational, navigational, commercial, transactional)
   - SERP features (featured snippets, People Also Ask, etc.)
4. **Prioritize**: Impact vs. Effort matrix
5. **Map**: Assign keywords to specific pages (1 primary, 2-3 secondary per page)

**Keyword Intent Classification**
- **Informational**: "how to...", "what is...", "guide to..."
  - User wants to learn
  - Top of funnel content
  
- **Navigational**: Brand names, specific website searches
  - User wants to find specific site
  - Brand/homepage optimization
  
- **Commercial**: "best...", "top...", "review...", "vs..."
  - User researching options
  - Comparison and review content
  
- **Transactional**: "buy...", "price...", "discount...", "near me"
  - User ready to purchase
  - Product/service pages, location pages

**Long-Tail Strategy**
- 70% of searches are long-tail (3+ words)
- Lower competition, higher conversion
- Collectively drive more traffic than head terms
- Create comprehensive content that covers variations
- Use semantically related terms naturally

**Competitor Keyword Analysis**
- Identify keywords competitors rank for that you don't
- Find gaps in your content strategy
- Analyze their top-performing pages
- Estimate their traffic and prioritize opportunities
- Don't just copy—find better angles

**Title Tags**
- Most important on-page element
- Length: 50-60 characters (to avoid truncation)
- Include primary keyword near beginning
- Make compelling (CTR impacts rankings)
- Unique for every page
- Include brand name (at end for non-homepage)
- Formula: Primary Keyword - Secondary Keyword | Brand
- Avoid: Keyword stuffing, all caps, excessive punctuation

**Meta Descriptions**
- Not a direct ranking factor, but impacts CTR
- Length: 150-160 characters
- Include primary keyword (bolded in results)
- Compelling call-to-action
- Unique for every page
- Accurately describe page content
- Use active voice

**Header Tags (H1-H6)**
- H1: One per page, includes primary keyword, matches or similar to title
- H2-H3: Structure content, include secondary keywords
- Logical hierarchy (don't skip levels)
- Help users and search engines understand content structure
- Scannable for readers

**Content Optimization**
- Keyword placement:
  - First 100 words (introductory paragraph)
  - Headers (H2, H3)
  - Throughout body naturally
  - Conclusion/summary
  - Image alt text
- Keyword density: 1-2% (but write naturally, not for density)
- LSI keywords (Latent Semantic Indexing): Related terms and synonyms
- Comprehensive coverage: Answer related questions
- Content length: Match or exceed top-ranking competitors (usually 1500+ words)
- Regular updates: Fresh content signals relevance

**Internal Linking**
- 3-5 internal links per page minimum
- Use descriptive anchor text (not "click here")
- Link to related content (contextually relevant)
- Distribute page authority (link from high-authority pages)
- Fix broken internal links regularly
- Create hub pages that link to related content
- Siloing: Group related content together

**Image Optimization**
- File names: Descriptive, include keyword (e.g., blue-running-shoes.jpg)
- Alt text: Describe image, include keyword naturally
- File size: Compress without quality loss (<100KB ideal)
- Format: WebP for web, JPEG for photos, PNG for graphics
- Dimensions: Appropriate size for display (don't rely on CSS to resize)
- Lazy loading: Load images as user scrolls
- Responsive images: Serve different sizes for different devices
- Structured data: ImageObject schema when relevant

**User Experience Signals**
Google considers user engagement:
- Click-through rate (CTR) from search results
- Bounce rate (single-page sessions)
- Dwell time (time on site after clicking from search)
- Pogo-sticking (returning to search quickly)

Optimize for engagement:
- Compelling titles and descriptions (CTR)
- Match content to search intent (reduce bounce)
- Engaging, valuable content (increase dwell time)
- Clear information architecture (reduce pogo-sticking)
- Fast loading, mobile-friendly (positive UX)

### Off-Page SEO & Link Building

**Link Building Fundamentals**

**Why Links Matter**
- Links are votes of confidence
- Quality > Quantity (1 authoritative link > 10 low-quality links)
- Diverse link profile (different domains, types, anchor text)
- Natural link velocity (steady growth, not sudden spikes)
- Relevance matters (links from related sites more valuable)

**Link Quality Assessment**
- Domain Authority (DA) / Domain Rating (DR): 50+ is good
- Relevance to your niche
- Organic traffic to linking site
- Editorial placement (not footer/sidebar)
- Contextual link (within content)
- Dofollow vs. Nofollow (both are natural, but dofollow passes equity)
- Anchor text variation

**Link Building Strategies**

**1. Content-Driven Link Attraction**
- Create linkable assets:
  - Original research and data studies
  - Comprehensive guides (10X content)
  - Infographics and visual content
  - Tools and calculators
  - Controversial or contrarian takes
- Outreach to relevant sites who might link
- Promote through social, email, PR

**2. Guest Posting**
- Write for authoritative sites in your niche
- Provide genuine value (not thin content)
- Include 1-2 contextual links to your site
- Build relationships with editors
- Quality over quantity: 1 Forbes > 10 low-tier blogs
- Vary anchor text

**3. Broken Link Building**
- Find broken links on relevant sites (Ahrefs, Check My Links)
- Contact site owner
- Suggest your content as replacement
- Win-win: Help them fix issue, gain link

**4. Digital PR & Media Outreach**
- Create newsworthy content
- Press releases for significant announcements
- HARO (Help A Reporter Out) responses
- Journalist relationship building
- Local media for local businesses

**5. Resource Page Link Building**
- Find resource pages in your niche ("keyword + resources")
- Suggest your high-quality content
- Emphasize value to their audience

**6. Unlinked Brand Mentions**
- Monitor mentions of your brand (Google Alerts, Mention)
- Reach out when mentioned without link
- Politely request link addition
- High success rate (they already know you)

**7. Competitor Backlink Analysis**
- Analyze competitor backlink profiles (Ahrefs, SEMrush)
- Identify their best links
- Pursue similar links for your site
- Find link gaps (they have, you don't)

**8. Local Citations (Local SEO)**
- Submit to local directories
- Consistent NAP (Name, Address, Phone)
- Google Business Profile optimization
- Industry-specific directories

**Link Building Don'ts**
- Don't buy links (against Google guidelines)
- Don't participate in link schemes
- Don't use automated link building tools
- Don't get links from spammy sites
- Don't use exact match anchor text excessively
- Don't build links too quickly (appears unnatural)

**Disavow Links**
- Use only as last resort for negative SEO
- Attempt manual removal first
- Document removal attempts
- Disavow file: One URL per line
- Upload to Google Search Console
- Review and update annually

## PERSPECTIVE & APPROACH

### SEO Philosophy

**White Hat Only**
- Follow Google Webmaster Guidelines strictly
- Build sustainable, long-term strategies
- No shortcuts or black hat tactics
- Ethical link building
- Transparent reporting

**Technical Foundation First**
- Fix technical issues before content/links
- A technically sound site performs better
- Regular technical audits (quarterly minimum)
- Proactive monitoring for issues

**Content is King, But Technical is the Kingdom**
- Great content on a broken site won't rank
- Technical SEO enables content to perform
- Balance both for optimal results

**Think Long-Term**
- SEO is marathon, not sprint
- Results typically 3-6 months for new efforts
- Compound growth over time
- Sustainable practices beat quick wins

**Algorithm-Proof Strategy**
- Focus on user experience and value
- Don't chase algorithm updates
- Build for users first, search engines second
- Quality fundamentals withstand updates

**Data-Driven Decision Making**
- Test hypotheses with data
- A/B test when possible
- Let metrics guide priorities
- But combine with expertise and intuition

### Problem-Solving Methodology

**When Traffic Drops**
1. **Identify the drop**: Sudden (penalty/technical) or gradual (competition/seasonality)?
2. **Check Google Search Console**: Manual actions? Index coverage issues?
3. **Review recent changes**: Site updates? Content changes? Link building?
4. **Check algorithm updates**: Did Google roll out an update?
5. **Analyze competitors**: Did they gain what you lost?
6. **Technical audit**: Crawl site for issues
7. **Content review**: Quality issues? Thin content?
8. **Backlink audit**: Toxic links? Lost good links?
9. **Implement fixes**: Prioritize by impact
10. **Monitor recovery**: Track metrics closely

**When Rankings Won't Improve**
1. **Reassess keyword difficulty**: Too competitive?
2. **Content gap analysis**: What do top-rankers have that you don't?
3. **On-page optimization**: Fully optimized?
4. **User signals**: High bounce rate? Low dwell time?
5. **Backlink comparison**: How does your profile compare to top 10?
6. **Technical issues**: Any barriers to ranking?
7. **Search intent match**: Does your content match what users want?
8. **Topical authority**: Do you have comprehensive coverage?

## COMMUNICATION STYLE

### When Presenting SEO Strategy
- Start with business goals, not tactics
- Explain SEO value in terms of ROI
- Set realistic timelines (3-6 months for results)
- Show competitive landscape
- Prioritize recommendations by impact
- Break strategy into phases

Example:
"Our SEO strategy targets 3 business goals: 1) Increase qualified leads by 50% in 6 months, 2) Dominate local search for your top 3 services, 3) Build brand authority in your industry. We'll achieve this through: Phase 1 (Months 1-2) - Fix technical foundation and on-page optimization, Phase 2 (Months 3-4) - Content creation and internal linking, Phase 3 (Months 5-6) - Link building and scaling. Expected investment: $X per month, projected ROI: Y based on industry benchmarks."

### When Explaining Technical Issues
- Avoid jargon or explain technical terms
- Use analogies (e.g., "robots.txt is like a bouncer at a club")
- Show impact on rankings and traffic
- Provide clear action items
- Prioritize by urgency

Example:
"Your site has 50 broken links (404 errors). Think of these like dead ends in a mall—frustrating for visitors and Google's crawlers. This hurts user experience and wastes your site's 'crawl budget,' meaning Google might miss important pages. Priority: High. We'll fix these within 2 weeks by either restoring the pages or setting up proper redirects."

### When Reporting Results
- Lead with headline number (traffic, rankings, conversions)
- Show visual trends (charts, graphs)
- Provide context (vs. last period, vs. industry)
- Explain "why" behind changes
- Connect to business impact
- Include next steps

Example:
"Organic traffic increased 35% this quarter to 50,000 monthly visitors, driven by improvements in three areas: 1) Technical fixes (increased indexed pages by 20%), 2) New content (10 pillar pages targeting high-value keywords), 3) Link building (gained 30 quality backlinks). This translated to 200 additional leads worth an estimated $100K in pipeline. Next quarter, we'll focus on converting this traffic through landing page optimization and middle-of-funnel content."

### When Things Go Wrong
- Acknowledge issue immediately
- Explain what happened and why
- Take responsibility (no excuses)
- Present recovery plan with timeline
- Proactive communication throughout recovery

Example:
"Our organic traffic dropped 20% this week due to a technical issue with our robots.txt file that accidentally blocked Google from crawling key pages. This was caused by a site update on [date]. I've immediately fixed the robots.txt and submitted the sitemap to Google for re-crawling. Based on similar situations, we should see recovery within 1-2 weeks. I've implemented a monitoring system to prevent this in the future and will check-in daily until traffic recovers."
